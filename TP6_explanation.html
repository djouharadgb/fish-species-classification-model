✅ TO DO #1 — Define the Neural Network Layers (__init__)

❌ Original:   # TO DO — no layers defined
✅ Replaced with: 4 hidden layers with Dense + LeakyReLU + BatchNorm + Dropout

Architecture chosen:

Flatten (128×128×3 = 49152 inputs)
    ↓
Dense(1024) → LeakyReLU → BatchNorm → Dropout(0.4)
    ↓
Dense(512)  → LeakyReLU → BatchNorm → Dropout(0.3)
    ↓
Dense(256)  → LeakyReLU → BatchNorm → Dropout(0.3)
    ↓
Dense(128)  → LeakyReLU → BatchNorm → Dropout(0.2)
    ↓
Dense(9, softmax)  ← output layer (9 fish classes)

Why this architecture?

ChoiceWhy
4 hidden layers (1024→512→256→128)Pyramid structure: starts wide to capture many features, gradually narrows to combine them. More layers = more capacity to learn complex fish patterns.
LeakyReLU instead of ReLUAvoids the "dying ReLU" problem — neurons can still learn even when their output is negative, because LeakyReLU allows a small negative slope (0.1).
BatchNormalization after each layerNormalizes the outputs of each layer so training is faster and more stable. Reduces sensitivity to the initial learning rate.
Dropout (0.4 → 0.3 → 0.3 → 0.2)Randomly disables neurons during training to prevent overfitting. Higher dropout early (0.4) where there are more neurons, lower at the end (0.2).

The code added:

self.dense1 = Dense(1024, dtype="float32")
self.relu1  = LeakyReLU(negative_slope=0.1)
self.bn1    = BatchNormalization()
self.drop1  = Dropout(0.4)

self.dense2 = Dense(512, dtype="float32")
self.relu2  = LeakyReLU(negative_slope=0.1)
self.bn2    = BatchNormalization()
self.drop2  = Dropout(0.3)

self.dense3 = Dense(256, dtype="float32")
self.relu3  = LeakyReLU(negative_slope=0.1)
self.bn3    = BatchNormalization()
self.drop3  = Dropout(0.3)

self.dense4 = Dense(128, dtype="float32")
self.relu4  = LeakyReLU(negative_slope=0.1)
self.bn4    = BatchNormalization()

✅ TO DO #2 — Forward Pass (call method)

❌ Original:   # TO DO — the layers weren't called, so nothing was computed
✅ Replaced with: Sequential call of all 4 layers in order

What is the call method?

In Keras, __init__ just defines layers. The call method is what actually runs them when you pass data through the model. Without filling the TO DO here, the model was just jumping from Flatten directly to Softmax — no learning was happening in between.

The code added:

def call(self, inputs, training=False):
    x = self.flatten(inputs)

    x = self.dense1(x)
    x = self.relu1(x)
    x = self.bn1(x, training=training)   # training=True during fit, False during predict
    x = self.drop1(x, training=training) # Dropout only active during training!

    x = self.dense2(x)
    x = self.relu2(x)
    x = self.bn2(x, training=training)
    x = self.drop2(x, training=training)

    x = self.dense3(x)
    x = self.relu3(x)
    x = self.bn3(x, training=training)
    x = self.drop3(x, training=training)

    x = self.dense4(x)
    x = self.relu4(x)
    x = self.bn4(x, training=training)
    x = self.drop4(x, training=training)

    outputs = self.classifier(x)
    return outputs

⚠️ Important detail: training=training

BatchNormalization and Dropout behave differently during training vs prediction:

LayerDuring trainingDuring prediction
DropoutRandomly disables neuronsAll neurons active (no dropout)
BatchNormUses batch statisticsUses learned running mean/variance

Passing training=training ensures this switch happens automatically.

✅ TO DO #3 — Data Augmentation (ImageDataGenerator)

❌ Original:   only rescale=1./255 — images normalized but no augmentation
✅ Added 6 augmentation techniques to the training generator

The code added:

train_datagen = ImageDataGenerator(
    rescale=1./255,           # normalize pixel values to [0, 1]
    rotation_range=20,         # randomly rotate ±20 degrees
    width_shift_range=0.15,    # shift image left/right by 15%
    height_shift_range=0.15,   # shift image up/down by 15%
    zoom_range=0.2,            # random zoom in/out 20%
    horizontal_flip=True,      # randomly mirror the image
    brightness_range=[0.8, 1.2], # vary brightness ±20%
    fill_mode='nearest'        # fill empty areas after rotation/shift
)

Why data augmentation?

The model sees the same 9000 images over and over during training. Without augmentation, it memorizes them instead of learning general fish features → overfitting.

Augmentation creates artificially varied versions of each image on the fly, so the model learns to recognize fish regardless of angle, position, or lighting.

TechniqueWhat it simulatesWhy useful for fish
rotation_range=20Camera tilted slightlyFish can be photographed at various angles
width/height_shiftFish not centeredSubject not always in the middle of photo
zoom_range=0.2Different distancesCamera may be closer or further from fish
horizontal_flipMirror imageFish facing left or right, both valid
brightness_rangeDifferent lightingUnderwater, flash, natural light vary

Note: Augmentation is only applied to the training generator. The validation and test generators only use rescale=1./255 — we always evaluate on clean, unmodified images.

🎯 Bonus Fix — Learning Rate

❌ Original:   Adam(learning_rate=0.01)
✅ Changed to: Adam(learning_rate=0.001)

0.01 is too high for a DNN on image data — the model overshoots the optimal weights and oscillates. 0.001 is the standard starting point for Adam on image classification tasks. The ReduceLROnPlateau callback will lower it further automatically if needed.
